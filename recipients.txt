# GitHub Changelog Digest — recipient list
#
# ⚠️  THIS REPOSITORY IS PUBLIC.
#     Anything you put in this file is committed to public git history forever.
#     Do NOT add real personal/work email addresses here while the repo is
#     public — keep those in the encrypted DIGEST_TO_EMAIL repository secret
#     instead (Settings → Secrets and variables → Actions). The digest reads
#     that secret automatically when this file has no addresses.
#     Only use this file for real recipients if the repository is private.
#
# To add a recipient:    append a line with their email and commit + push.
# To remove a recipient: delete the line, or comment it out with a leading `#`.
#
# Rules:
#   - One email address per line.
#   - Lines starting with `#` are ignored.
#   - Blank lines are ignored.
#   - A trailing comment is allowed if separated by whitespace then `#`,
#     e.g.  alice@example.com   # alice — paused while travelling
#
# Examples:
#   frank@example.com
#   alice@example.com
#   # bob@example.com           # off the list — re-enable when he's back
